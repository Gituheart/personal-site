## 为什么做这个

作为技术人，每天需要关注 AI、前端、移动端等领域的动态。但信息源太分散——HuggingFace、GitHub Trending、Hacker News、各种公众号——逐个打开太费时间，不看又怕错过重要进展。

需要一个工具：**帮我跑一遍所有信息源，筛掉垃圾，留下干货，排好版**。

---

## 架构设计

```
主 Agent (调度) → SubAgent A (抓取 Tier1) 
                → SubAgent B (抓取 Tier2)
                → Browser Worker (JS 渲染页面)
                          ↓
                 AI 筛选 + 评分 (1-5⭐)
                          ↓
                 Markdown 日报 + HTML 精美版
```

### 三层抓取策略

| 层级 | 方式 | 适用场景 |
|------|------|---------|
| Tier 1 | WebFetch | 静态页面、RSS、API |
| Tier 2 | WebFetch + 解析 | 需要提取结构化数据的页面 |
| Browser | Headless 浏览器 | JS 渲染的页面（如 HuggingFace） |

### 智能筛选规则

抓取到的内容不是全部收录，而是经过 AI 评估：

- 筛入：前沿技术、高深技术、提效技术、实用资讯
- 筛除：泛科普、营销软文、过度学术化、招聘帖
- 每条内容 1-5 星评分，只收录 3 星以上
- 目标：每日精选约 20 条高质量内容

---

## 输出格式

每条内容包含：
- 标题和摘要
- 3 个要点提炼
- 来源链接
- 关键词标签
- 质量评分（1-5⭐）

生成两种格式：
1. **Markdown 日报**：纯文本，适合存档和二次编辑
2. **Apple 风格 HTML**：精美网页，适合分享和阅读
