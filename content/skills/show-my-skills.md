## 为什么做这个

装了 156 个 Agent Skill，却发现大部分从来没被触发过。问题出在哪？

经过一番排查，发现 Agent Skill 的触发机制本质上是**关键词匹配**——你说的话要"命中"技能 description 里的关键词，它才会被加载。很多技能的 description 写得很模糊，甚至是空的，自然永远不会被触发。

于是我做了这个"元技能"：**一个管理所有其他技能的技能**。

---

## 它能做什么

### 技能全览

扫描三个技能目录（Cursor / Agent / Claude），提取每个技能的：
- 名称和所属目录
- 功能描述（做什么用）
- 触发关键词（怎么触发它）
- 使用次数

### 按使用频率排序

用得越多的排越前面。从来没用过的排在最后——**你一眼就能看出哪些技能在吃灰**。

### 使用频率追踪

每次任意技能被成功调用，自动递增该技能的使用计数，持久化到 `usage.json`。用久了就是一份个性化的技能使用画像。

### 关键词筛选

可以按关键词筛选，比如问「有哪些飞书相关的技能」，只展示匹配的技能。

---

## 怎么用

| 你说的话 | 它做的事 |
|---------|---------|
| "我的技能" / "技能列表" | 展示全部技能（按使用频率排序） |
| "有哪些飞书技能" | 筛选展示飞书相关技能 |
| "show my skills" | 同上 |

---

## 技术实现

一个 Python 脚本（`show_skills.py`），做三件事：

1. **扫描**：遍历 `~/.cursor/skills/`、`~/.agents/skills/`、`~/.claude/skills/` 下所有 SKILL.md
2. **解析**：从 YAML frontmatter 提取 name/description，从正文提取触发关键词（只从 description 和明确的触发声明中提取，过滤掉代码示例中的噪音）
3. **排序**：读取 `usage.json` 中的使用计数，按频率降序排列

---

## 诞生背景

这个技能诞生于一次"技能大清洗"——我把 156 个技能精简到了 50 个，删掉了 106 个冗余、概念性、未配置的技能。清洗过程中意识到：**装多少技能不重要，知道自己有什么、怎么触发它们才重要**。于是做了这个技能来解决"装了不知道有"的问题。
