## 我们遇到了什么问题

技术支持团队每周处理大量工单，在日常工作中面临几个痛点：

**质检耗时费力**

- 每周需要抽检工单，逐条打开、阅读对话、对照质检标准打分，一个人检 20 条工单轻松花掉半天
- 质检标准有 20 项，涉及严谨性、沟通表达、灵活性、跟进及时性、工具使用等多个维度，人工逐条比对容易遗漏
- 不同人质检尺度不一致，难以标准化

**工单价值沉淀困难**

- 大量工单处理完就"沉睡"了，其中 BUG、需求建议、可复用的解决方案（wiki）、文档问题等有价值信息没有被系统性地整理
- 想要回顾"最近一周有哪些 BUG"或"哪些问题可以沉淀为 FAQ"，只能靠人工一条条翻
- 工单打标签不全，靠标签统计覆盖不了全部工单

**流程繁琐**

- 从工单系统导出数据需要手动操作浏览器、复制 JSON、保存文件
- 质检结果要写报告、推企微、通知相关人，整个链路断裂在多个工具之间

## TicketClaw 能做什么

TicketClaw 是一个 AI Agent 技能，用自然语言对话即可完成工单质检和分类归纳的全流程，**从取数到报告到推送，一句话搞定**。

### 功能一：工单质检

对指定范围的工单逐条自动检测，按团队质检标准（20 项规则、合格线 >90 分、一票否决项）打分，输出精美的质检报告网页并推送到企微群。

**你只需说：**

> "质检 3/9 移动端App 的工单"

**它会自动：**

1. 打开浏览器 → 你扫码登录一次 → 自动拉取当天全部工单数据
2. 逐条对照质检标准检测（工具使用、回复相关性、一票否决项等）
3. 生成质检报告网页（合格/不合格、分数、扣分项、改进建议）
4. 推送到企微群

### 功能二：工单分类归纳

对指定范围的工单自动分类为 **BUG / 需求 / wiki / 文档 / 不归类**，生成分类报告网页、问题记录表，并将 wiki 类工单自动整理为可复用的 FAQ 文档。

**你只需说：**

> "归纳 2/27 到 3/5 移动端App 的全部工单"

**它会自动：**

1. 拉取全部工单数据（一次登录）
2. 优先看标签 → 无标签时用关键词速判决策树自动分类
3. 生成精美的分类报告网页（按 BUG/需求/wiki/文档 分表展示）
4. wiki 类工单自动整理为完整的 markdown 文档（含问题环境、解决方案、是否可优化）
5. 推送报告到企微群

### 功能三：一句话触发，智能判断意图

如果你只给了工单号，没说要做什么：

> "T202603050182, T202603050183"

它会先问你：**「您希望对这批工单做质检，还是做分类/归纳分析？」** ——不会擅自执行。

## 如何使用

### 第一步：安装（一次性）

```bash
# 安装 Python 依赖
pip install playwright
playwright install chromium

# 配置企微推送（可选，不配则用默认群）
export WECOM_WEBHOOK_KEY="你的企微群机器人key"
```

### 第二步：把技能放到 Claude Code

将 `TicketClaw` 文件夹复制到 `~/.claude/skills/` 下：

```
~/.claude/skills/ticketclaw/
├── SKILL.md       ← Agent 识别技能的入口
├── README.md      ← 详细技术文档
├── *.py           ← 运行代码
├── *.md           ← 质检/分类规则文档
├── 问题类型请求参数.json
└── favicon_b64.txt
```

### 第三步：在对话中使用

打开 Claude Code，在对话中直接说：

| 你说的话 | 它做的事 |
|---------|---------|
| "质检 3/9 移动端App 的工单" | 拉取 → 质检 → 生成报告 → 推企微 |
| "质检 T202603050182" | 拉取该工单 → 质检 → 报告 |
| "归纳 2/27 到 3/5 的全部工单" | 拉取 → 分类为 BUG/需求/wiki/文档 → 报告 + wiki |
| "T202603050182" | 先问你：质检还是归纳？ |
| "质检 3/1 到 3/7，Soc 产品开发" | 按问题类型筛选 → 质检 |

首次运行时会弹出浏览器，扫码登录工单系统即可（**一次任务只需登录一次**）。

## 输出示例

### 质检报告网页

- 顶部统计：工单总数、合格数、不合格数
- 筛选：可按合格/不合格筛选
- 每条工单：是否合格、分数、AI 回复评估、扣分项明细（可展开）与改进建议

### 分类归纳报告网页

- 顶部统计卡片：BUG / 需求 / wiki / 文档各多少条
- 四个分类表格：工单号、问题摘要、判定理由
- wiki 类工单可展开查看完整的归纳 markdown（工单号、环境、解决方案、是否可优化），支持一键复制

两种报告都会以 HTML 文件推送到企微群，下载打开即可查看。

## 常见问题

**Q: 浏览器弹出后没有反应？**

登录成功后页面需加载工单列表，脚本通过监听请求获取 token。如果 90 秒内页面没发起请求，会超时退出，重新运行即可。

**Q: 分类结果大量"不归类"？**

无标签工单的分类依赖对话中的关键词匹配，对于进行中或简单咨询类工单会判为不归类。在工单系统中打标签（线上bug / 需求建议 / 沉淀FAQ / 文档不清晰）可以显著提高分类准确率。

**Q: 如何修改质检标准？**

质检规则在 `qc_engine.py` 中实现，对应的标准说明在 `默认质检需求.md`。如需调整合格线或扣分项，修改这两个文件即可。

**Q: 如何修改分类规则？**

分类规则在 `classify_engine.py` 中实现，对应的规则文档在 `工单分类与归纳规则.md`。标签映射表和关键词速判决策树都可以按需调整。

**Q: 企微推送到哪个群？**

通过环境变量 `WECOM_WEBHOOK_KEY` 配置。每个人设置自己群的机器人 key 即可。不设置则推送到默认群。
